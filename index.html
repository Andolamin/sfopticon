SfOpticon

In your Salesforce controlling your versions

Download this project as a .zip file Download this project as a tar.gz file

SfOpticon is a tool to add software conifguration managment capabilities to the Salesforce development cycle.

Getting SfOpticon

The source for SfOpticon can be downloaded on this website as a zip archive, or a gzipped tarball. However, if you would always like to be up to date, it is recommended that you clone the git repository. 

$ git clone https://github.com/sfopticon/sfopticon.git

For more information, or for even more download methods, please visit https://github.com/sfopticon/sfopticon

Configuration

Configuration details are stored in the 'application.yml' file. Please fill out your database and scm details here. Currently, MySQL and Github are the only supported configurations.

Installation

Installation is simple and requires ruby and bundler 

$ cd sfopticon/
$ bundle install

If installation fails, it is most likely due to missing database drivers. Please see the troubleshooting section for more details.

Usage

First, a production environment must be setup.

$ ./bin/ennironment.rb create --name <YOUR_PROD_ENV_NAME> --production

After you have created your production environment, feel free to create other non-production environments as you see fit.

The next step is to create a changeset using the scanner tool. You can do this manually, but it is recommended to create a relatively frequent (~20 - 30 minutes) cron job for this. As its name implies, the scanner tool scans your environment for changes and synchronizes them with the remote SCM system.

$ ./bin/scanner.rb changeset --org <ENVIRONMENT_TO_SCAN>

Lastly, the integration tool allows you to integrate branches, and then deploy the merged changes to any environment.

$ ./bin/integrate.rb merge --soruce <SOURCE_ENV> --destination <DEST_ENV>

This will perform the merge and stash your changes in a temporary branch, allowing you to deploy at your leisure.
